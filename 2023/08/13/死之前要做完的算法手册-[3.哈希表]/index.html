哈希表基础

哈希表也称散列表，是根据Key直接访问地址的数据类型

哈希函数

利用函数将元素的数据取得一个哈希值，这个哈希值也即要存放的地址

哈希碰撞

当存放的地址已经有值的时候就发生了哈希碰撞，处理哈希碰撞有以下几种方法

  • 线性探测法：从当前位置往下遍历找到一个空位置将数据存进去
  • 链式寻址法：链式寻址法也即拉链法，插入到位置结点的下一个结点中，形成链表
  • 再哈希法：用另外的哈希函数计算一个新的哈希值

快乐数

编写一个算法来判断一个数 n 是不是快乐数。

「快乐数」 定义为：

  • 对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和。
  • 然后重复这个过程直到这个数变为 1，也可能是 无限循环 但始终变不到 1。
  • 如果这个过程 结果为 1，那么这个数就是快乐数。

如果 n 是 快乐数 就返回 true ；不是，则返回 false 。

题解

这道题用双指针法，因为哈希法对去重的逻辑麻烦不好理解，而双指针比较下更易于做去重的操作

思路

a+b+c==0

首先，先对数组进行排序，这样能保证顺序是有序的方便做去重

1.对数组排序

2.用一层for循环,i从0开始 来找到第一个元素nums[i] 如果nums[i]>0 那么就可以return 因为a>0 排序后就不会有a+b+c==0

3.用left指针和right指针分别表示第二个元素和第三个元素，初始化left=i+1,right=nums.length-1;

4.考虑a元素去重，如果i>0 && nums[i]==nums[i-1] 那么continue

为什么不是nums[i]==nums[i+1]

原因是这个nums[i]==nums[i+1]判断的是结果集的不可重复，而元素如果为 -1 -1 2 第一个元素和第二个元素是可以相同的 相反，如果是判断和前一个元素相同因为b和c没变，a移动之后还是相同的元素，所以这个结果集不能算在内

5.判断a+b+c?0 如果＞则移动right– 如果<则移动left++如果为= 则添加到result中

6.添加完之后移动指针，这里考虑b和c的去重 如果nums[left]==nums[left+1] 持续left++ 如果nums[right]==nums[right-1] 持续right–
