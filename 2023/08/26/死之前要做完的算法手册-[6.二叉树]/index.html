平衡二叉树

平衡二叉树是指左右子树的高度差不超过1，就是平衡二叉树

满二叉树

满二叉树是指一个二叉树只有度为0的叶子结点和度为2的非叶子结点，除了叶子结点的每个结点都有左右子树

完全二叉树

完全二叉树是指除了最后一层是满二叉树，并且最后一层必须都先填满左边

二叉搜索树

二叉搜索树是指结点有数值，二叉搜索树是一个有序树

平衡二叉搜索树

平衡二叉搜索树是指结点有数值的平衡二叉树，左右子树高度差不超过1

二叉树的递归遍历

二叉树分为深度优先搜索和广度优先搜索，我们这里先探究的是深度优先遍历中的利用递归遍历

深度优先搜索，当某一个方向遍历到最后一个结点则结束当前方向，进行下一个方向的遍历

广度优先搜索，当遍历完一层之后再遍历下一层，一层一层搜索

二叉树的递归遍历主要分为：前、中、后序遍历

前序遍历的遍历方式为:中左右

中序遍历的遍历方式为:左中右

后序遍历的遍历方式为:左右中

所以我们记前中后序遍历只要知道这是在说明中间结点的位置

上面我们提到了二叉树的深度优先搜索中的递归遍历方法，而递归遍历通常也能用非递归的方法解决

迭代遍历：用一个栈来模拟二叉树的遍历过程

迭代遍历主要的是处理两件事：遍历结点以及处理结点

前序遍历

Leetcode.144题(easy)

思路

我们前序遍历也即是中左右的遍历方式，而栈是一个先进后出的一个数据结构

在这里我们遍历结点和处理结点都是同时的，也即当我们遍历到中结点就要将中结点数据添加list中

而左右子结点入栈应该是先右子树入栈再是左子树入栈，因为LIFO的特性，先处理左子树那就得后入栈

实现

①将根节点添加进栈中

②将中结点弹出，如果结点非空则添加到结果集合中，并且将结点的右子树和左子树入栈

（入栈顺序为中，右，左。处理顺序为中，左，右）

③当处理结束将结果集返回

后序遍历

Leetcode.145题(easy)

思路

后序遍历也即左右中的遍历方式 ，我们前序遍历的遍历方式是中左右，然后先入栈右子树再入栈左子树就能达到处理顺序是中左右。而我们将子树的入栈顺序颠倒，也即入栈左子树再入栈右子树就可以得到中右左的结果集，我们最后再将中右左的结果集反转就能得到左右中的遍历结果。

也即 左右中 == 前序遍历 中左右 ->将子树入栈顺序颠倒 中右左 -> 反转结果集 左右中

实现

②将中结点弹出，如果结点非空则添加到结果集合中，并且将结点的左子树和右子树入栈

（入栈顺序为中，左，右。处理顺序为中，右，左）

③反转结果集(中，右，左 反转成 左，右，中)

中序遍历

Leetcode.94题(easy)

思路

这里遍历结点和处理结点不是同时进行的，也即当我们遍历到这个结点要先去处理左节点，所以我们引入了一个用于遍历的指针

通过指针的不断移动确认当前遍历到的结点，如果当前指针不为空则将结点入栈，再去操作左右子树

如果当前指针为空 说明上一个结点没有左子树 则返回到上个结点将其出栈，再去判断右子树是否为空

实现

①定义一个用于遍历的指针cur，代表着此时遍历到的结点

②如果当前结点不为空 则入栈，将指针指向当前结点的左子树

③如果当前结点为空，则说明左子树为空，返回上一个结点将其弹出实现了左中的路线，再将当前结点指向右子树

④如果当前指针为空并且栈为空说明遍历已经结束 实现了左中右的逻辑

思想：以中序遍历为假设

当当前结点不为空的时候，将其左右子树和当前结点重新入栈 需要实现左中右的逻辑

  • 假如左右子树都不为空 那么实现栈中 右结点 当前结点 null 左结点

  • 假如左子树为空右子树不为空， 那么栈中 右结点 当前结点 null

  • 假如左子树不为空右子树为空， 那么栈中 当前结点 null 左结点

  • 假如左右子树都为空，那么栈中 当前结点 null

当遇见null结点的时候只要pop栈就能找到当前结点，也即维护了一个当前结点后跟着一个空指针标记

首先我们先要理解，树的高度和树的深度的意义：

  • 高度： 某个结点到子叶结点的距离
  • 深度：某个结点到根节点的距离

如图所示一颗二叉树的深度和高度的关系

那么本题就能知道其实根节点的高度也是最大深度

我们通过递归来求树的深度的话，我们选用哪种递归方式比较好求？ 答案是后序遍历，因为后序遍历是左右中，可以获取到左右子树的高度再+1就能知道本结点的高度。

实现

  • 明确递归的参数以及返回参数：需传入结点，返回高度

  • 明确递归的结束条件： 结束当遍历到NULL结点时，返回高度为0 那么就能保证子叶结点高度为1

  • 单层遍历要做的： 遍历获取左子树和右子树的高度，然后取最大值+1 就代表本结点的高度

利用完全二叉树的特性

思路

  • 因为完全二叉树除了最后一层其他都是满二叉树，所以如果一个二叉树是满二叉树的话 那它的结点个数为2^n -1 个，n为左右子树的深度，而获取左右子树的深度则只需要遍历左子树外侧和右子树外侧
  • 如果一个树不是满二叉树的话，那么统计左右两边子树的个数

实现

  • 递归遍历的返回值和参数：返回值是结点数，参数是传入的根节点

  • 递归遍历的结束条件：

    • 如果root==null 说明是空节点，上一个结点是叶子结点，返回0

    • 如果左子树深度和右子树深度一致，说明是满二叉树，返回2^n-1个结点数(利用位运算 2<<n -1)

      • 如何获取左右子树深度

      • 左子树深度，定义一个left指向左子树，然后一直往左子树指，当非空深度+1

      • 右子树深度，定义一个right指向右子树，然后一直往右子树指，当非空深度+1

  • 单层递归的逻辑：

    说明这棵完全二叉树不是满二叉树，那么统计左右子树的节点个数，最后返回左右子树结点数之和+1

这里获取左右子树深度只搜索子树外侧，不搜索内侧

题解

思路：遍历方式选用前序遍历(中左右)，因为父节点要指向子结点所以采用前序遍历，同时递归的时候要进行一个回溯操作。

实现

  • 选用前序遍历，定义一个result用于装结果集，定义一个path用于表示单层路径

  • pathGet方法用递归

    • 递归的参数和返回类型：递归参数path：单层路径，result：结果集，root：结点
    • 递归的结束条件：当root是叶子结点即可以收获结果集，添加路径到result中。也即叶子结点是left==null&&right==null
    • 递归的单层逻辑：非叶子结点
      • 如果左子树不为空则向左收集路径，递归结束调用remove删除最后一个元素进行回溯
      • 如果右子树不为空则向右收集路径，递归结束后调用remove删除最后一个元素进行回溯

  • 注意点

    • path要在判断是否为叶子结点之前收集root的数据，因为调用方法时root为叶子结点就结束了，不可能传入空结点所以不用判空。
    • 当向左向右递归结束之后都要回溯数据：删除path路径的最后一个数据，也即左右子树数据
    • path中的数据是存放的数据，如1->3->5存放到path中是135，所以要进行一个拼接再添加到结果集中
