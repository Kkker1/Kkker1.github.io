回溯算法理论基础

回溯法又称回溯搜索法，是一种暴力搜索法

回溯的本质就是穷举，穷举所有可能，选出想要的结果

回溯法可以抽象成一个N叉树，在集合递归中寻找结果，而集合长度为树的宽度，递归的深度构成了树的深度

77.组合(优化)

给定两个整数 n 和 k，返回范围 [1, n] 中所有可能的 k 个数的组合。

你可以按 任何顺序 返回答案。

Leetcode.77题(medium)

题解

在这里我们进行一个剪枝的优化

假设

假如n=4，k=4的时候，我们可以简单知道唯一的答案就是[1,2,3,4]。而我们之前的穷举除了要查找1，还要查找2、3、4，但是这个时候无论如何都不能达到需要的k值，也即我们当集合中剩余的元素不可能达到组合的个数的时候就可以进行剪枝

也即我们可以求进行组合搜索的时候至多索引的位置

实现

  • 获取至多索引的长度

    **k-list.size()**为还需要的元素个数

    n-(k-list.size())+1为至多索引的位置，我们可以假设极端情况来判断这个是否要加1

    假设n为4 k为3 list中元素为0 所以4-(3-0)+1 也即最多索引为2

题解

实现

  • 全局变量

    • List<List> result :用于统计结果集
    • List path :用于记录单个路径
    • int[] used :用来标记这个位置元素是否使用

  • 单层递归的处理逻辑

    • 去重处理

      • 判断当(i!=0 && candidates[i-1]==candidates[i] && used[i-1]==0时，直接跳过不处理

      • i！=0 第一个元素不进行判断

      • candidates[i-1]==candidates[i] 说明当前元素和前一个元素值相同，要准备进行去重(如果是层次上的相同则去除，如果是深度上的相同则保留)

    • 添加路径到path中，将used[i]置为1

    • 递归搜索i+1的位置

    • 回溯，回溯path也回溯used[i]，将used[i]置为0，表示这个位置元素没有用过

      • 也即!used.contains（i）&&!set.contains(ticket) 说明这个机票没被用过

        • 如果result.size()==1，说明是刚起飞，判断当前机票ticket是否起始站为”JFK”，如果不是则continue
        • 如果result.size()！=1，判断当前机票ticket是否起始站和result中最后一个元素相等，如果不相等则continue
        • result添加目的站
        • used添加索引i，set添加当前机票
        • 进行递归
          • if(backtracking(tickets)){ return true},如果递归中返回true就不需要继续搜索了
        • 回溯

      • 当循环遍历到最后位置则返回false
