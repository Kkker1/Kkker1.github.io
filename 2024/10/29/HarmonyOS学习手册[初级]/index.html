HarmonyOS官网

简介

为了应对应用开发目前所遇到的机遇、挑战和趋势，HarmonyOS孕育而生。

而当今移动应用开发所遇到的主要挑战

  • 针对不同设备上的不同操作系统，重复开发，维护多套版本
  • 多种技术栈，对人员技能要求高
  • 多种开发框架，不同的编程范式
  • 命令式编程，关注细节，变更频繁，维护成本高

HarmonyOS是新一代的智能终端操作系统，为不同设备的智能化、互联与协同提供了统一的语言，为用户带来简捷，流畅，连续，安全可靠的全场景交互体验

HarmonyOS的核心技术理念

一次开发，多端部署

多端开发环境

  • 多端双向预览
  • 低代码可视化开发

多端开发能力

  • 多端UI适配
  • 事件交互归一
  • 设备能力抽象

多端分发机制

  • 多设备按需分发
  • 多入口按需分发

可分克合，自由流转

可分可合

  • 独立开发元服务，按需组合为应用
  • 应用和元服务独立商家

自由流转

  • 跨端迁移
  • 多端协同

统一生态，原生智能

统一生态

  • 多应用生态共存，支持多端融合
  • 统一接入标准，实现快速接入

原生智能

  • 服务能力开放
  • 应用能力开放
  • 芯片能力开放

DevEco Studio搭建

下载安装包

先下载安装包，DevEco Studio集成了SDK、构建插件、ohpm等工具，所以可以直接使用

官方下载地址

傻瓜式安装

下载解压，打开可执行文件(exe文件)，修改文件安装路径，傻瓜式下一步点击即可。

配置环境，启动第一个项目

总所周知，第一个项目都是Hello World

快速入门，启动一个Hello World项目

新建项目

新建一个Empty Ability项目，然后配置项目文件，命名为Hello World工程

初始化环境

初始化项目，如果报错可以通过检查开发环境来调整，通过Diagnose Development Environment也即诊断开发环境。

Diagnose 具体打开路径： Help -> Diagnostic Tools -> Diagnose Development Environment

如果出现如下报错

解决方案

1、将c盘下的.npmrc文件的register由淘宝镜像改成华为云镜像 https://repo.huaweicloud.com/repository/npm

具体路径为：C:\Users\Administrator\.npmrc

2、清除Node.js缓存

3、清除HarmonyOS缓存，并且重启应用

File -> Invalidate Caches.. -> invalidate and Restart

4、也许是Node.js文件权限不足，打开Node.js文件夹，将权限改成完全控制

启动项目

从项目文件中找到Index.ets 然后选择设备启动（无绑定设备可以用Previewer启动项目），启动第一个Hello World工程，并且可以通过按钮交互

快捷键使用

快捷键 用法
Ctrl+Alt+B 跨语言跳转(如使用Native的C/C++函数)
Ctrl+Alt+L 格式化代码
Ctrl+F12 打开代码结构树
Alt+F7 查看变量引用位置
“/**”+回车键 文档注释

Ctrl+H 查看父子类的层次结构
Ctrl+O 重写方法
Alt+Insert 重构构造函数、GET/SET方法、生成声明信息到Index文件

模拟器

当没有真机运行时，我们可以自定义模拟器(虚拟机)来模拟真机。

在菜单栏的Tools ->Device Manager 通过Edit设置模拟器的存储路径，然后点击New Emulator新建一个模拟器，新建模拟器时需要下载模拟器镜像，然后点击Next设置内存和存储空间，再点击Finish完成模拟器创建

  • 模拟器运行

    模拟器通过点击Actions中的运行(RUN)按钮，随即等待开机即可运行

  • 清除用户数据

    模拟器通过点击Actions中的Wipe User Data清除用户数据，类似于回复出厂设置

  • 关闭模拟器

    模拟器可以通过显示的×或者Actions中的暂停(STOP)按钮关闭运行

  • 操控屏幕（缩放）

    可以通过按住Ctrl+鼠标左键缩放模拟器屏幕，按住Ctrl键之后，鼠标点击则会改成缩放状态

  • 安装应用和上传文件

    文件上传可以直接使用拖拽的方式，这样即可安装应用、上传文件

数据类型

这里目前只介绍有哪些数据类型以及声明方法，具体数据类型具有的属性或者方法需要再由其他文章介绍

Number

任何整数、浮点数都能由number表示。

  • 整数部分

    • 十进制

    • 十六进制

    • 八进制

    • 二进制

  • 浮点数

    • 小数点

    • 十进制小数

Boolean

布尔类型，只有true和flase两个值。通常赋值为flase，程序运行阶段赋值为true

String

字符串类型，同样可以在字符串中利用转义字符\来表示字符

  • 单引号之间的字符可以表示字符串

  • 双引号之间的字符可以表示字符串

  • 可以用反引号(`)之间的字符可以表示字符串

Void

空返回类型，常用于指定某函数的返回为空

Object

对象类型，是所有类型的基类型，所有类型的值可以直接被赋给Object类型变量，都会被自动装箱

Array

数组类型，由赋值的数据类型组成的存储对象。

Enum

枚举类型，预先定义好值的值类型。

Union

联合类型，由多个类型组成的引用类型。联合类型包括了变量可能所有的类型

有点像是java中的父类和子类的概念，父类包括了子类的所有类型

Aliases

匿名类型，为数组、函数、对象字面量、联合类型提供名称，或为已有类型提供替代名称

比较运算符

运算符 说明
=== 如果两个操作数严格相等（对于不同类型的操作数认为是不相等的），则返回true。
!== 如果两个操作数严格不相等（对于不同类型的操作数认为是不相等的），则返回true。
== 如果两个操作数相等，则返回true。
!= 如果两个操作数不相等，则返回true。
> 如果左操作数大于右操作数，则返回true
>= 如果左操作数大于或等于右操作数，则返回true。
< 如果左操作数小于右操作数，则返回true。
<= 如果左操作数小于或等于右操作数，则返回true。

位运算符

运算符 说明
a & b 按位与，a和b的二进制对应位都为1时，将这个位设为1，反之设为0
a | b 按位或，a和b的二进制对应位有一个为1时，将这个位设为1，反之设为0
a ^ b 按位异或，a和b的二进制对应位如果不同，将这个位设为1，反之设为0
~ b 按位非，反转操作数的位（按64位计算，如10010 则反转之后为FFFF FFFF FFFF FFED)
a << b 左移，将a的二进制按位左移b位（如10010 << 2 则左移之后为 0100 1000）
a >> b 算数右移，将a的二进制按位右移b位，保持原正负属性（即正数高位补0，负数高位补1）
a >>> b 逻辑右移，将a的二进制按位右移b位，高位补0

逻辑运算符

运算符 说明
a && b 逻辑与
a || b 逻辑或
！a 逻辑非

continue

我们也拿之前For循环的语句举例，当我们倒数为3的时候就跳到下一次循环中(2)

函数

函数声明

举例

传参规则：

  • 每个参数必须标记类型
  • 如果参数是可选参数，则调用函数时可以省略传递该参数
  • 可以为可变参数(rest参数)，那么可变参数(rest参数)一定是最后一个参数

可选参数

可选参数意思是指定过程中参数是可选的，可以不传递则默认为undefined

可选参数的格式为：name ?: type

  • name 变量名
  • ?: 表示参数为可选参数
  • type 参数数据类型

举例

Rest参数

可变参数(rest参数)，表示这个参数有零个或者多个数量

rest参数的格式为：... name : types

  • ... 表示该参数为可变参数
  • name 参数名
  • : 后接参数类型
  • types 接变量数据类型的数组

举例

返回类型

  • 显式返回类型
  • 不标注返回类型

  • 不需要返回类型，使用void或者省略标注

作用域

函数的作用域也和基础语言一致，都有局部属性

函数内被定义的变量和实例仅可以在函数内部访问，而不能从外部访问。并且当函数的局部变量和外部有重名，则会函数内的局部定义将会覆盖外部定义直到结束函数。

函数调用

函数调用如果有传入参数，则实参会被赋给形参。

完整形式

  • 具体解析

    • (x : number , y: number) 表示函数有两个number 类型参数x和`y``
    • ``:` 后接返回数据类型

    • => 表示此为lambda 形式函数，后接函数体

    • } 大括号(右)，表示这是函数体的结束

  • 省略返回数据类型

  • 省略大括号、省略return
